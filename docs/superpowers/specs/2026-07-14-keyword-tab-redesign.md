# 关键词搜索 Tab 小红书参数独立化 设计文档

## 1. 目标

将关键词搜索 Tab 中**仅适用于小红书**的参数（每个关键词N条、评论数/条、排序方式、内容类型）从通用配置行中分离出来，放到独立的「小红书参数」区块，避免用户误以为这些参数影响微博搜索。

## 2. 改动范围

仅修改 `social_monitor_gui.py` 中 `_build_tab_keyword()` 方法，不涉及后端脚本逻辑。

## 3. 旧布局（问题）

```
┌─ 搜索配置 ────────────────────────────┐
│ 产品线:[下拉框]  ...  [取消全选]        │
│ [关键词 checkbox 网格]                 │
│ 自定义关键词:[输入框]  [+ 添加]         │
│                                        │
│ ☑ 微博  ☑ 小红书  每个关键词:[5]条      │ ← 混在一起
│ 评论数/条:[300]条   排序方式: ...       │ ← 实际只影响小红书
│ 内容类型: ...                          │ ← 实际只影响小红书
│                                        │
│ ☑ 发送到 AI Agent                      │
└────────────────────────────────────────┘
```

## 4. 新布局

```
┌─ 搜索配置 ────────────────────────────┐
│ 产品线:[下拉框]  ...  [取消全选]        │
│ [关键词 checkbox 网格]                 │
│ 自定义关键词:[输入框]  [+ 添加]         │
│ ───────────────────────────────────── │
│ 搜索平台:  ☑ 微博    ☑ 小红书          │
│                                        │
│ ── 小红书参数 ──                       │
│ 每个关键词:[5]条  评论数/条:[300]条     │
│ 排序方式: ○最多点赞 ○最多评论           │
│ 内容类型: ○不限 ○仅图文                │
│ ───────────────────────────────────── │
│                                        │
│ ☑ 发送到 AI Agent                      │
└────────────────────────────────────────┘
```

### 改动点

1. **搜索平台**从原行拆分到独立行，加 `搜索平台:` 标签
2. **「小红书参数」用 `LabelFrame` 包裹**，标题明确标注这是小红书独有参数
3. 参数控件内容不变，仅调整位置和容器
4. 微博下面没有参数，清爽直观

## 5. 文件改动清单

| 操作 | 文件 | 说明 |
|------|------|------|
| EDIT | `social_monitor_gui.py:_build_tab_keyword()` | 重组 UI 布局（约 lines 418-473） |

无其他文件变更。`keyword_search.py` 参数逻辑不变。

## 6. 验证

1. Tab 正常显示，微博/小红书勾选正常
2. 「小红书参数」LabelFrame 明确可见，且其内的控件值正确传入 `keyword_search.py`
3. 不选小红书时，那些参数无影响（与现有行为一致）
