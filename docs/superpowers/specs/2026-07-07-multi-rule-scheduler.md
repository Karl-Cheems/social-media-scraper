# 多规则定时任务 设计文档

## 需求

将当前单频率定时任务改为**多条独立规则**的调度系统。

- 每条规则 = 一个动作（热搜 / 关键词搜索 / 账号监控）+ 独立的调度计划
- 频率改为星期多选 + 具体时间点（不再分每天/每周两个模式）
- 规则列表展示所有已设规则，支持启用/禁用、立即执行、删除
- 规则持久化到 JSON 文件，重启 GUI 后恢复

## 数据模型

```python
ScheduleRule = {
    "id": str,              # uuid
    "name": str,            # 用户自定义名称，如"工作日热搜"
    "enabled": bool,        # 是否启用
    "action_type": str,     # "hot" | "keyword" | "account"
    "params": dict,         # 动作特定参数（见下方）
    "weekdays": list[int],  # [0..6], 0=周一
    "time": str,            # "HH:MM"
    "send_agent": bool,     # 执行后发 AI Agent
}
```

### 各动作的 params

| action_type | params 字段 |
|---|---|
| `hot` | `platform` (weibo_hot/weibo_ent/douyin/merged), `limit` (int) |
| `keyword` | `product_line` (str), `per_keyword` (int) |
| `account` | `urls_file` (str), `limit` (int) |

## 持久化

- 保存到 `BASE_DIR/schedule_rules.json`
- GUI 启动时自动加载
- 添加/删除/修改规则时即时写入

## 调度逻辑

### 校验逻辑
添加规则时检查：
- 名称不能为空
- 至少勾选一个星期
- 时间格式正确

### 轮询线程
- 每 30 秒检查一次
- 遍历所有 `enabled=True` 的规则
- 判断：当前 weekday 在规则 weekdays 中 **且** 当前时间 >= 规则 time **且** 同一分钟还未执行过
- 命中则执行

### 防重复执行
- 每条规则记录 `last_run_date`（日期字符串 YYYY-MM-DD）
- 同一日期、同一规则不重复执行

### 执行方式
- 每条规则在独立线程中执行
- 调用现有 `_run_script_common` 方法
- 执行完成后更新 UI 中的上次执行时间+状态
