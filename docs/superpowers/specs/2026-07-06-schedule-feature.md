# 定时任务功能 设计文档

> **For agentic workers:** After user approval, implement per this spec.

## 需求

GUI 的「⏰ 定时任务」tab 目前只是占位，需求是：
1. **三个功能都可选**：热搜、关键词搜索、账号监控 — 各自有 checkbox
2. **每个功能保留原有的配置参数**：热搜选平台+数量、关键词选关键词+平台+数量、账号监控选 URL 列表
3. **频率可定制**：每天跑（指定几点）、每周跑（指定星期几+几点）
4. **发送到 AI Agent 可选**：每个功能独立选择
5. **总开关**：开启/关闭定时任务
6. **跑完后自动发送到 AI Agent**

## 架构

### GUI 设计

定时任务 tab 分为两块：

**上半部分（配置区）：**
```
┌─────────────────────────────────────────────┐
│ ⏰ 定时任务配置                              │
│ ┌─────────────────────────────────────────┐ │
│ │ [☑] 开启定时任务                        │ │
│ │ 频率: [每天 ○ 每周 ○]                    │ │
│ │ 时间: [09:00 ▼] (Spinbox 时:分)          │ │
│ │ 星期: [一 ☑ 二 ☑ 三 ☑ 四 ☑ 五 ☑ 六 ☑ 日 ☑] (仅每周模式) │
│ └─────────────────────────────────────────┘ │
│                                              │
│ ┌─────────────────────────────────────────┐ │
│ │ ☑ 热搜  │ 平台: [合并模式] │ 数量: [15] │ │
│ │ ☑ 关键词搜索 │ 产品线: [元气森林] │ 平台: ☑微博 ☑小红书 │ │
│ │ ☑ 账号监控 │ URL文件: [urls.txt] │ 已选: N个 │ │
│ └─────────────────────────────────────────┘ │
│                                              │
│ [▶  启动定时任务]                            │
│                                              │
│ ┌─────────────────────────────────────────┐ │
│ │ 日志区域                                 │ │
│ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

### 运行机制

- 启动定时任务后，用 `self.after()` 或 `threading.Timer` 定时检查
- 到点时，用现有 `_run_script_common()` 方法运行对应的脚本
- 脚本运行完后会自动触发 notify_agent.py 发送
- 运行日志写入定时任务的日志区域

定时任务本身不需要新的 CLI 参数，它只是 GUI 层面的一个定时调度器。
